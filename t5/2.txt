Задание
2) Имеется программа (исходный код которой приводится ниже,
компилировать с ключами: -fno-stack-protector -no-pie). Вам необходимо
произвести анализ программы с помощью отладчика для выяснения длины
массива для ввода пароля и адреса ветки условия проверки корректности
ввода пароля, которая выполняется при условии совпадения паролей.
Ввести пароль (строку символов)таким образом, чтобы перезаписать адрес
возврата на выясненный адрес (есть символы которые нельзя ввести с
клавиатуры, поэтому можно использовать перенаправление ввода(<) при
запуске программы).

Исходный код

#include <stdio.h>
#include <string.h>
#include <stdlib.h>

int IsPassOk(void);

int main(void)
{
    int PwStatus;
    puts("Enter password:");
    PwStatus = IsPassOk();
    if (PwStatus == 0) {
        printf("Bad password!\n");
        exit(1);
    } else {
        printf("Access granted!\n");
    }
    return 0;
}

int IsPassOk(void)
{
    char Pass[12];
    gets(Pass);
    return 0 == strcmp(Pass, "test");
}

строка 0x1400015ac (jne): Это команда "прыгнуть, если не равно" (Jump if Not Equal).
она спрашивает: "результат проверки пароля НЕ равен 0?"

если ДА (пароль верный) => процессор прыгает по адресу, указанному в команде => 0x1400015c7.
если НЕТ (пароль неверный) => процессор не прыгает и идёт дальше вниз => на строку 0x1400015ae (где Bad password).


строка 0x1400015c7:
сюда попадает процессор, если пароль верный.
здесь загружается строка "Access granted!".
вывод: Целевой адрес для перезаписи = 0x1400015c7.


теперь нужно понять, через сколько байт от начала ввода пароля находится адрес возврата.
адрес возврата лежит в стеке выше, чем локальные переменные.

смещение = (размер буфера) + (размер сохранённого указателя RBP)

12 (буфер) + 8 (RBP) = 20 байт
